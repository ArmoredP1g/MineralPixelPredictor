# MineralPixelPredictor 工程项目详细说明

## 1. 项目目标

本项目用于高光谱像素级铁品位反演，当前工程同时包含两条实验主线：

1. 深度学习主线：基于 1D 自编码器和回归头进行两阶段训练。
2. 传统机器学习主线：对 SVR、PLSR、XGB、RF、LR 等基线方法做统一对比。

项目的核心目标不是只给出一个模型，而是建立一套能够复现实验、管理检查点、输出结构化对比结果的实验框架。

## 3. 数据流与训练流程

### 3.1 深度学习主线

深度学习实验是标准两阶段流程。

#### 阶段 A：表征学习（AE）

输入为采样得到的像素光谱，先进入编码器，再进入解码器重构光谱。

训练目标有两种：

1. 基线 AE：仅使用 MSE 重构损失。
2. 对比学习 AE：使用 `MSE + lambda * SupCon`。

#### 阶段 B：回归训练（Predictor）

将阶段 A 的编码器输出送入 Predictor，回归样品品位。

当前实现默认在阶段 B 冻结编码器，只训练回归头。

### 3.2 传统基线主线

传统基线流程是：

1. 样本级数据聚合。
2. 随机搜索超参。
3. 训练模型。
4. 评估样本级与像素聚合指标。
5. 输出 JSON / HTML 对比结果。

## 4. 当前支持的消融实验

[ablation_fold0.py](ablation_fold0.py) 中定义了以下实验矩阵：

| 编号 | 名称 | 编码器 | 是否使用 SupCon | 说明 |
| --- | --- | --- | --- | --- |
| #1 | Baseline | StandardCAE | 否 | 标准 1D-CAE，纯 MSE |
| #2 | ASPP-CAE | ASPP_CAE | 否 | 多尺度空洞卷积编码器 |
| #3 | Mixer-AE | MixerAE | 否 | 固定 patch 的 MLP-Mixer 编码器 |
| #4 | BestEncoder + GradeWeightedSupCon | 自动从 #2/#3 选最优 | 是 | 在最优编码器上加入 SupCon |

其中 #4 的逻辑不是手工指定，而是脚本在 #2 和 #3 完成后，按 `test_sample.rmse` 选择更优编码器。

## 5. 检查点与输出文件设计

### 5.1 原始训练入口

原始 [train.py](train.py) 使用全局 `session_tag`，所有 fold 保存到：

- `ckpt/<session_tag>/encoder_fold{i}_step{N}.pt`
- `ckpt/<session_tag>/fold{i}_step{N}.pt`

### 5.2 新消融入口

新增 [ablation_fold0.py](ablation_fold0.py) 后，每个实验使用独立 session：

- `ckpt/<session_base>_exp1_fold0/`
- `ckpt/<session_base>_exp2_fold0/`
- `ckpt/<session_base>_exp3_fold0/`
- `ckpt/<session_base>_exp4_fold0/`

每个目录中保留：

- `cfg.json`
- `encoder_fold0_best.pt`
- `encoder_fold0_last.pt`
- `fold0_best.pt`
- `fold0_last.pt`

这比老版本“只保留一个 step checkpoint”更利于复现实验与回溯分析。

### 5.3 输出报告

消融入口最终输出：

- `ablation_fold0_final.json`
- `ablation_fold0_final.html`

传统基线入口最终输出：

- `comparison_fold0_results.json`
- `comparison_fold0_results.html`

## 8. 正式实验参数说明

这一节是最终参数说明的重点，后续出结果时建议直接引用本节。

### 8.1 与老版本对齐的部分

以下参数已经按老版本完整训练口径设置：

| 参数 | 正式实验值 | 老版本来源 | 是否对齐 |
| --- | --- | --- | --- |
| `ae_steps` | `40000` | `ae_step_per_fold` | 是 |
| `pd_steps` | `150000` | `pd_step_per_fold` | 是 |
| `learning_rate` | `0.001` | `learning_rate` | 是 |
| `lr_decay` | `0.98` | `lr_decay` | 是 |
| `lr_decay_step` | `1000` | `lr_decay_step` | 是 |
| `lr_lower_bound` | `1e-8` | `lr_lower_bound` | 是 |
| `num_workers` | `6` | `num_workers` | 是 |

### 8.2 需要额外说明的部分

以下配置不是老版本 `train.py` 显式存在的参数，或者语义与老版本不完全一致，因此必须在实验说明里写清楚。

#### 1. `samplepoint=500`

这是训练时每次从一个样本中随机抽取的像素数。

在数据集类中，该参数直接决定单个样本在一个训练 step 中包含多少个像素光谱。

#### 2. `--experiments 1 2 3 4`

这是新增消融脚本特有的实验矩阵控制参数。

老版本没有统一消融入口，因此不存在完全一一对应项。

#### 3. `session_base=ablation_fold0_final`

这是新增实验命名空间，用于隔离不同实验的输出目录。

老版本只使用单一 `session_tag=final`。

#### 4. `tau=0.07`

这是 SupCon 温度系数，仅对 #4 生效。

老版本没有对比学习分支，因此这个参数属于新增实验超参。

#### 5. `lambda_supcon=0.1`

这是 SupCon 损失权重，仅对 #4 生效。

老版本没有该项，属于新增实验超参。

#### 6. `mixer_patch_size=12`

这是 Mixer-AE 的固定 patch 长度。

对应 168 波段输入时，会形成 14 个 patch。

#### 7. `ae-loader-batch`

脚本默认 `ae-loader-batch=4`，但只有 SupCon 路径会实际使用这个值。

具体逻辑见 [ablation_fold0.py](ablation_fold0.py)：

- 如果 `use_supcon=False`，AE DataLoader batch size 强制为 `1`
- 如果 `use_supcon=True`，AE DataLoader batch size 使用 `--ae-loader-batch`

这意味着：

- #1、#2、#3 在 AE 阶段仍然按单样本批次跑
- #4 在 AE 阶段为了构造跨样本正负关系，会使用多样本批次

因此，`ae-loader-batch` 不属于老版本配置，而是 SupCon 引入后新增的必要参数。

### 8.3 正式实验建议在论文/报告中明确写出的最终参数

建议最终说明中写成下面这种形式：

> 深度学习消融实验在 fold0 上进行。基础训练参数与原始 CAE-MLP 版本保持一致：AE 训练步数为 40000，回归头训练步数为 150000，学习率为 0.001，学习率衰减系数为 0.98，衰减步长为 1000，学习率下限为 1e-8，DataLoader 工作进程数为 6。每次从单个样本中随机采样 500 个像素用于训练。新增对比学习实验中，温度参数 tau 设置为 0.07，损失权重 lambda 设置为 0.1。Mixer-AE 采用固定 patch 切分，patch size 为 12。

## 9. 目前工程的优点与限制

### 9.1 优点

1. 已经具备完整的深度学习与传统基线双路线实验能力。
2. 已支持结构化 JSON 和 HTML 输出，适合论文整理。
3. 已支持每实验独立 checkpoint 保存，避免结果互相覆盖。
4. 已支持自动挑选最优编码器进入 SupCon 实验。

### 9.2 当前限制

1. 当前新增消融入口默认只面向 fold0，而不是完整 5 折。
2. SupCon 的最终表现对 `ae-loader-batch`、`lambda_supcon`、`tau` 可能比较敏感。
3. 正式实验时间较长，建议在稳定 GPU 环境中运行。
4. 当前报告里主要给出 RMSE、R2、像素均值/中位数指标，还没有补充 t-SNE 等可视化分析。

---

## 10. 建议的后续维护方向

1. 将 fold0 消融脚本扩展到多 fold 批量执行。
2. 将正式实验参数写入单独配置文件，而不是全部依赖命令行传参。
3. 在 HTML 报告中增加训练配置摘要与 checkpoint 路径索引。
4. 增加 t-SNE / 特征可视化产物输出。
